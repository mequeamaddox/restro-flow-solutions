To build a reliable, vendor-agnostic invoice parser that works across all kinds of invoice formats — especially for restaurant inventory — you need to combine a layered strategy that mimics what QuickBooks, Dext, and FoundBankApp do under the hood.

Here's a breakdown of parsing strategies, algorithms, and structure analysis techniques that work well in production-ready OCR invoice pipelines:

🧠 HIGH-LEVEL STRATEGY
🔁 Multi-Pass Parsing Pipeline

Rather than trying to extract everything in one pass, use staged passes:

Metadata Pass → extract vendor, date, invoice number

Block Grouping Pass → isolate line item regions

Line Scoring Pass → identify potential product lines

Pattern Matching & Fallbacks → extract or infer missing fields

Postprocessing → normalize, validate, dedupe, and convert

🗂️ STRUCTURE ANALYSIS TECHNIQUES
1. Visual Clustering / Table Detection

If using Textract:

Use FeatureTypes: ["TABLES", "FORMS"] to identify rows and bounding boxes

Group words by rowIndex and columnIndex when available

For text-only OCR (like Tesseract): simulate table structure using line spacing and indentation

💡 Tip: If two lines have similar x-positions and vertical spacing, treat them as part of the same “row”.

2. Line Classification with Heuristics + Probabilistic Scoring

Give each line a score based on:

Contains price (decimal or integer)

Contains food-related or product keywords

Near other item lines

Not blacklisted (e.g. “subtotal”, “invoice”, “thank you”)

Example scoring:

lineScore = hasPrice ? 1.0 : 0
           + hasKeyword ? 0.8 : 0
           + nearOtherItems ? 0.5 : 0
           - isHeaderOrTotal ? 1.5 : 0


Only parse lines above a confidence threshold.

3. Regex + Fuzzy Extraction

Use layered regex to extract:

Dates: \d{2}[\/\-\.]\d{2}[\/\-\.]\d{2,4} or \d{4}-\d{2}-\d{2}

Invoice Numbers: /invoice[^\n]*?(\d{4,})/i

Quantities: (\d+)\s*(lbs?|cases?|ct|ea|each|gal|g|kg)

Units: Look for unit tokens after quantity or in column headers

Use fuzzball.js or similar to match fuzzy product names if you have a product dictionary.

🔍 PATTERN MATCHING TECHNIQUES
4. Sliding Window Line Matching

When one line contains only a description and the next has the price, use a sliding window approach:

for (let i = 0; i < lines.length; i++) {
  if (isLikelyDescription(lines[i]) && isLikelyPrice(lines[i+1])) {
    createLineItem(lines[i], lines[i+1])
  }
}

5. Contextual Field Association

Map price fields based on relative position:

End of line → likely total price

Middle column → unit price

First column → quantity

Near product word → likely belongs to the same item

🔄 FALLBACK STRATEGIES
6. Rule-Based Matching with Fallback Priority

If you can't detect all 4 fields (desc, qty, unit price, total), do:

description + total_price → assume quantity = 1

description + quantity + total → infer unit price

description + unit_price → infer total if next line has qty

Use previous row structure if current row is ambiguous

🧠 ADVANCED OPTIONS (Optional)
7. Train a Line Item Classifier

Train a simple ML model (e.g., decision tree or even GPT) to classify lines as:

item, header, footer, junk, date, vendor, etc.

Use features like:

Line length

Word count

Price pattern presence

Known product word presence

Capitalization pattern

8. Pre-trained LayoutLM or Donut (Document Understanding Models)

If you want full-blown AI:

Use Donut (no OCR required, outputs JSON directly)

Or LayoutLMv3 with HuggingFace to classify and extract fields based on document layout

Great for scalability, but may require more compute and labeled training data.

✅ BEST PRACTICE COMBO (No ML Required)

If you want a universal parser without ML, here's the best combo:

Layer	Strategy
Metadata	Regex + top 10 line scan
Vendor	Uppercase + top position match
Line items	Heuristic scoring + regex + proximity window
Price parsing	Integer-to-decimal conversion fallback
Quantity/unit	Regex patterns for quantity + optional header inference
Line grouping	Simulated rows via spacing or bounding boxes
Validation	Require at least 2 of 3 (desc, unit price, total) before creating row
